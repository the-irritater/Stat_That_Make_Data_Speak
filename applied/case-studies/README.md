## What Makes a Good Case Study

Every case study in this repo follows this structure:

1. **Business Question** — A real decision someone needs to make
2. **Data** — Documented source, cleaned, reproducible
3. **Analysis** — Step-by-step with code and commentary
4. **Key Visual** — One chart that tells the story
5. **Key Finding** — One shareable sentence with a specific metric
6. **Business Recommendation** — What should the company actually do?
7. **LinkedIn Post Draft** — Ready to share, with GitHub link
