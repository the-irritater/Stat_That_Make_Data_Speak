# Day 35: Clean Data vs Messy Data

## LinkedIn Post Copy
```text
Day 35 of Stats That Make Data Speak

Topic: Clean Data vs Messy Data

Clean data is organized, consistent, complete, and ready for analysis.

Messy data contains problems.

Examples:

Missing values
Duplicate rows
Wrong spellings
Mixed date formats
Incorrect data types
Extreme values
Unclear column names

Real-life example:

In a student dataset, "Mumbai", "mumbai", "Mum", and "Bombay" may all refer to the same city.

But a computer treats them as different values.

That is messy data.

Data analytics use:

Most real-world analysis time is spent cleaning data, not building models.

Bad data leads to bad charts, bad models, and bad decisions.

Key takeaway:

Clean data makes analysis reliable.

Messy data makes even advanced methods useless.
```
