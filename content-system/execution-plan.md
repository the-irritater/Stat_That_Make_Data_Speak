# Execution Plan — Stats That Make Data Speak

## 8-Week Portfolio Execution Plan

This plan turns the Stats That Make Data Speak series into a visible, recruiter-ready data analytics portfolio.

---

## Core Rule

Every piece of work must answer one question:

**What decision does this analysis help someone make?**

A notebook is not complete when the code runs. A notebook is complete when it has:

- [x] A real dataset
- [x] One clear business or research question
- [x] Cleaned data
- [x] One strong visual
- [x] One insight with a number, comparison, or surprise
- [x] One decision-focused takeaway
- [x] A LinkedIn post linked to GitHub

---

## Weekly Output Requirements

Each week must produce:

- [ ] One strong applied notebook
- [ ] One lighter analysis or mini notebook
- [ ] One shareable visual
- [ ] One LinkedIn post with the visual shown inside the post
- [ ] One GitHub link
- [ ] One decision-focused takeaway

Every two weeks → one full case study.

## Insight Quality Rule

A weak insight:
> Students who study more perform better.

A stronger insight:
> Students studying more than 3 hours scored around 18% higher on average, but the variation remained high. This suggests study time matters, but it is not the only factor affecting performance.

Every insight must include at least one of:
- A number
- A comparison
- A surprise

## Daily Filter

Before posting anything, check:

```text
Does this show that I can think with data?

If not, improve it.
```

Recruiters do not care that you learned pandas. They care whether you can take messy data, find a pattern, explain the risk, and support a decision.
